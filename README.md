# 2019年下期目標管理概要

## 目標管理課題

git、ソースツリー等のツールを用いながら
参考書「はじめてのAngular4」に沿ってAngularを学習、
サンプルプログラムの作成を行う。

## プロジェクトの目標

1か月に2度業務終了後に時間を設け、スケジュールに沿って
学習する。

## スケジュール

以下のスケジュールで進めていく。
スケジュールより、時間がかかっても理解を優先する。

* 4月 git等環境構築、第1章：Angularとは
* 5月 第2章：「Angularプロジェクト」を開始
* 6月 第3章：「Angular」の動く仕組み
* 7月 第4章：独自の「コンポーネント・クラス」
* 8月 第5章：操作に応答する
* 9月 第6章：ルーティング

## 反省点

* 決められたスケジュールに沿って目標を遂行できなかった。

* 環境構築等にかかる時間の見積もりを誤った。

* 学習内容自体は楽しんでできる内容ではなかった(モチベーションにつながる目標ではなかった)。


## 良かった点、業務に活かせた点

* コマンドが通らない事が多々あり、それによりファイルの階層、プロキシ等をよく意識するようになった。

* 自宅での環境構築を行ったことにより、nodeや環境変数の設定、コマンド等の知識がついて業務に活かせた。

* 業務で違う言語を扱ったことによって、javascriptで実現できる事(他の言語との違い)などをしっかり意識することができた。


## 総括

半期の目標管理を終えて、業務に活きたことがいくつもありましたが、
その中でも特に苦手な環境周りに活かすことができました。

この目標管理での一番の反省点は、
・8月以降スケジュールに沿って学習を進められなかったこと

原因としては
・作業環境を自宅に作っていなかったこと
・積極的に上司に質問しなかったこと

これを受けて、
・次回から学習する際は自宅PCで作業する(自宅PCで学習することで環境周りも学習できる)
・slackなどのツールを積極的に利用
・私生活で困ったことも相談
・内容を軽くさらって工数見積もりを行う
以上のように対策します。
